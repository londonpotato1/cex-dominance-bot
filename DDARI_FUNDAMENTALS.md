## 1. 따리의 본질

### 1.1 핵심 공식
```
따리 수익 = 입금액 vs 거래량의 싸움

거래량 > 입금액  →  김프 형성 (흥따리)
거래량 < 입금액  →  가격 하락 (망따리)
```

### 1.2 손바뀜 비율
- **정의**: 거래량 ÷ 입금액 = 손바뀜 횟수
- **예시**: 입금액 4억, 5분 거래량 75억 → 약 19번 손바뀜 → 초대박따리 (AVAIL 사례)
- **해석**:
  - 손바뀜 횟수 ↑ → 가격 급등 가능성 ↑
  - 손바뀜 횟수 ↓ → 가격 횡보/하락 가능성 ↑

### 1.3 김치프리미엄의 원리
```
매수세 > 매도세  →  🥬 김치 프리미엄 형성
매수세 < 매도세  →  ⬇ 가격 하락, 역프
```

**김프 발생 조건**:
1. 🇰🇷 국내에서 과매수 (예: 신규상장, 큰 호재)
2. 🌐 해외에서 급락 (예: LUNA 사태)
3. 🌐 해외 특정 코인 급등으로 역프 발생

## 2. GO/NO-GO 판단 기준 (상장 전)

> **핵심**: 상장 전 가격은 없다. GO/NO-GO는 **가격 외 요소**로 판단해야 한다.

### 2.1 GO 조건 (참여 권장)

| 요소 | GO 조건 | 확인 방법 |
|------|---------|----------|
| **핫월렛 물량** | 적음 (20억 미만) | Etherscan, Arkham, DeBank |
| **DEX 유동성** | 500k 이하 | DexScreener, GMGN |
| **현선갭** | 낮음 (5% 이하) | DataMaxi+, 거래소 직접 비교 |
| **네트워크 속도** | 느림 (POW, 자체메인넷) | 체인 Explorer |
| **브릿지 난이도** | 어려움/복잡함 | 직접 테스트 |
| **직전 상장** | 흥행 | 상장 복기 데이터 |
| **시황** | 상승장, 거래량 활발 | 업비트 거래대금 |
| **상장 시간대** | 저녁 시간대 (퇴근 후) | 공지 확인 |
| **업비트 기대감** | 빗썸 단독 → 업비트 기대 | 공지 패턴 분석 |

### 2.2 NO-GO 조건 (패스 권장)

| 요소 | NO-GO 조건 | 이유 |
|------|------------|------|
| **핫월렛 물량** | 무한 (바이낸스 등) | 입금액 폭증 예상 |
| **DEX 유동성** | 1M 이상 | 후따리 물량 충분 |
| **현선갭** | 높음 (10%+) | 헷징 비용 과다 |
| **네트워크 속도** | 빠름 (솔라나, 베이스) | 후따리 쉬움 |
| **브릿지** | 쉬움/불필요 | 입금 난이도 낮음 |
| **직전 상장** | 망따리 | 시장 심리 냉각 |
| **시황** | 하락장, 거래량 저조 | 매수세 부족 |

### 2.3 빠른 판단 체크리스트

```markdown
□ 핫월렛 물량 확인 (20억 미만?)
□ DEX 유동성 확인 (500k 이하?)
□ 현선갭 확인 (5% 이하?)
□ 네트워크 속도 확인 (느린 체인?)
□ 브릿지 필요 여부 확인 (복잡?)
□ 에어드랍 클레임 상태 확인 (잘 안됨?)
□ 직전 상장 결과 확인 (흥행?)
□ 상장 시간대 확인 (저녁?)
```

## 3. 공급 예측 요소

### 3.1 핫월렛 물량

**정의**: 거래소가 즉시 출금 가능한 코인 보유량

**확인 방법**:
1. **Etherscan** (EVM 체인): 코인마켓캡 → 코인 검색 → Explorers → Holders
2. **Arkham** (intel.arkm.com): 코인 검색 → 거래소별 보유량
3. **DeBank** (debank.com): 지갑 주소 → EVM 체인 통합 조회

**거래소별 핫월렛 특징**:
| 거래소 | 물량 수준 | 특징 |
|--------|----------|------|
| 바이낸스 | 무한 | 거의 항상 충분 |
| 바이비트 | 준수 | 종종 부족 |
| OKX | 준수 | 종종 부족 |
| 게이트/비트겟 | 적음 | 자주 부족 |

### 3.2 네트워크 속도

**체인별 속도 비교**:
| 체인 | 속도 | 특징 |
|------|------|------|
| 이더리움 | 중간 (~7분) | 36컨펌, 일정함 |
| 솔라나 | 빠름 (수 초) | 입금 몰리면 느려짐 |
| 베이스/옵티미즘 | 느림 | L2 특성 |
| 자체메인넷 (POW) | 매우 느림 | CKB, MINA 등 |
| Cosmos 계열 | 빠름 | IBC 지원 |

**POW vs POS**:
- **POW**: 트랜잭션 몰리면 체인 멈춤 가능 (CKB, MINA 사례)
- **POS**: 상대적으로 안정적

### 3.3 브릿지 난이도

**브릿지가 필요한 경우**:
- 바이낸스 알파 (BSC) → 국내 지원 체인 (ETH 등)
- 자체메인넷 → EVM 체인

**주요 브릿지**:
| 브릿지 | URL | 특징 |
|--------|-----|------|
| Stargate | stargate.finance | 범용 |
| Wormhole | portalbridge.com | 범용 |
| Synapse | synapseprotocol.com | 특정 토큰 |
| Orbiter | orbiter.finance | L2 특화 |
| CCTP | cctp.money | USDC 전용 |

### 3.4 DEX 유동성

**확인 사이트**:
- **DexScreener**: dexscreener.com
- **GMGN**: gmgn.ai

**기준값 (2025년 4분기 기준)**:
- 500k 이상: 입금액 충분 예상
- 200k 이하: 흥따리 or 상장 연기 가능성

### 3.5 현선갭

**정의**: 현물가격 - 선물가격의 차이

**공급에 미치는 영향**:
- 현선갭 ↑ → 헷징 비용 ↑ → 따리꾼 참여 ↓ → 입금액 ↓
- 현선갭 30%+ → 왠만하면 푸는 것이 유리

## 5. 선따리 vs 후따리

### 5.1 선따리 (Pre-Listing Arbitrage)

**정의**: 상장 전 물량을 확보하여 상장 시점에 매도

**전략**:
1. 상장 예상 코인 사전 분석
2. 핫월렛, DEX 유동성 파악
3. 헷징 (현물 매수 + 선물 숏)
4. 국내 거래소로 입금
5. 상장 시점에 김프에 매도

**장점**:
- 가격 변동 리스크 최소화 (헷징 시)
- 초기 물량 확보로 높은 수익 가능

**단점**:
- 헷징 비용 (현선갭, 수수료)
- 물량 구하기 경쟁 치열

### 5.2 후따리 (Post-Listing Arbitrage)

**정의**: 상장 후 김프 형성 시 추가 물량 입금

**전략**:
1. 상장 후 김프 모니터링
2. 김프 유지 시 추가 물량 매수
3. 빠른 입금 (체인 속도 고려)
4. 김프에 매도

**핵심 요소**:
- 후따리 유동성이 너무 많으면 → 후따리 안하거나 소량만
- 유동성 500k 이하로 적당하면 → 김프 죽을 때까지 계속

**체인별 입금 속도**:
- 솔라나: 빠름 (입금 몰리면 느려짐)
- 이더리움: 일정 (~7분)
- 베이스/옵티미즘: 느림

### 5.3 전략 비교

| 구분 | 선따리 | 후따리 |
|------|--------|--------|
| 타이밍 | 상장 전 | 상장 후 |
| 리스크 | 헷징으로 최소화 | 가격 변동 리스크 |
| 수익률 | 안정적 | 변동적 |
| 필요 역량 | 사전 분석, 물량 확보 | 빠른 실행, 입금 속도 |
| 적합 상황 | 물량 구하기 어려운 코인 | 김프 유지되는 코인 |

## 7. 새로운 개발 로드맵

### Phase 1: 기초 데이터 수집 (1-2주)

**목표**: 무료/저렴한 API로 핵심 데이터 수집

1. **Etherscan API 연동**
   - ERC-20 토큰 Holders 조회
   - 거래소 핫월렛 라벨링
   
2. **DexScreener API 연동**
   - DEX 유동성 조회
   - 페어 정보 조회

3. **거래소 선물 API 연동**
   - 바이낸스, 바이비트 선물가격
   - 현선갭 계산

### Phase 2: GO/NO-GO 예측 시스템 (2-3주)

**목표**: 상장 전 GO/NO-GO 자동 판단

1. **상장 공지 모니터링**
   - 업비트/빗썸 공지 크롤링
   - 텔레그램 알림 연동

2. **코인 분석 자동화**
   - 핫월렛 물량 자동 조회
   - DEX 유동성 자동 조회
   - 네트워크 정보 수집

3. **GO/NO-GO 스코어링**
   - 각 요소별 가중치 적용
   - 종합 점수 산출
   - 권장 행동 제시

### Phase 3: 실시간 모니터링 (3-4주)

**목표**: 상장 후 실시간 따리 기회 포착

1. **입금량 추적**
   - 핫월렛 주소 모니터링
   - CIELO 봇 연동 or 자체 구현

2. **거래량/김프 실시간 추적**
   - 상장 시점부터 1분/5분 거래량
   - 김프 변화 추적

3. **후따리 알림**
   - 김프 유지 + 입금량 저조 시 알림
   - 최적 진입 타이밍 제시

### Phase 4: 상장 복기 자동화 (4-5주)

**목표**: 모든 상장 데이터 자동 기록

1. **상장 데이터 자동 수집**
   - 입금액, 거래량, 김프, 시총 상승분
   - 흥/망 판정 자동화

2. **패턴 분석**
   - 흥따리 공통점 분석
   - 망따리 공통점 분석

3. **예측 모델 개선**
   - 과거 데이터 기반 모델 튜닝
   - 정확도 추적

### 제약사항

| 제약 | 대안 |
|------|------|
| Arkham API 유료 | Etherscan 무료 + 수동 핫월렛 라벨링 |
| DeBank API 유료 | DexScreener 무료 활용 |
| 일부 온체인 데이터 접근 불가 | 텔레그램 봇 (CIELO) 연동 |

| 선따리 | 상장 전 물량 확보 전략 |
